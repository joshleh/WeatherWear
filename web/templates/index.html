How it works

WeatherWear is a utility-based AI agent built for CSE 150A at UC San Diego. It fetches real weather data, classifies conditions with a machine-learning model, then applies a rule-based outfit policy. Here's exactly how each recommendation is made.

1. Weather data

Real-time and forecasted daily weather is fetched from the Open-Meteo API (free, no API key). Four features are collected for each day:

  • 💧 Precipitation (mm)
  • 🌡️ Temp max (°C)
  • 🌡️ Temp min (°C)
  • 💨 Wind speed (m/s)

2. ML weather classification

A Random Forest classifier (400 trees, balanced subsampling) trained on Seattle historical weather data (2012–2015) predicts a weather label from the four features:

  • ☀️ Sun
  • 🌧️ Rain
  • 🌦️ Drizzle
  • ❄️ Snow
  • 🌁 Fog

3. Outfit policy — top & bottom

Based on the average temperature (avg of max + min):

Avg tempTopBottom
≥ 22 °CShort-sleeve shirtShorts
14 – 21 °CT-shirt / light long-sleevePants
6 – 13 °CLong-sleeve shirtPants
< 6 °CWarm base layer + long-sleeveWarm pants

4. Outfit policy — outerwear

Outerwear is chosen by the predicted label and raw feature values:

Snow🧥 Insulated waterproof jacket
Wind ≥ 7.5 m/s (no rain/snow)🧥 Windbreaker
OtherwiseNone needed

6. Explanation

A template-based engine (no LLM) generates a conversational summary from the predicted label, raw weather features, and the outfit chosen. This keeps the demo 100% free with no API keys.
